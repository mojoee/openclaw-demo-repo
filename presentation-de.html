Was ist OpenClaw?

Erweiterbarkeit durch Plugins

Erweitern ohne Kerncode zu ändern

  • Channel-Plugins — Teams, Matrix, Mattermost
  • Memory-Plugins — vector stores, knowledge graphs
  • Tool-Plugins — custom capabilities beyond bash/browser
  • Provider-Plugins — custom LLM providers, self-hosted models

How it works

  • Plugins live in extensions/
  • Discovery via openclaw.extensions in package.json
  • Validated against TypeBox schemas
  • Hot-loaded when configured

Meet Pi

The agent powering OpenClaw

  • Built by Infellip — the core AI runtime
  • Agent-nativ design — tools, memory, reasoning
  • Persistent context — remembers your workspace
  • Multi-model support — Claude, GPT, local models

Why it matters

Pi isn't just a chat wrapper. It's designed from the ground up for agentic workflows — tool use, memory search, session management, and streaming responses.

This is what makes OpenClaw different from a chatbot — it's an agent that acts.

Fun fact: "Pi" stands for Personal Intelligence — your agent, your context, your tools.

End-to-End Message Flow

Vom User-Message zum Response

  • User → Channel Adapter → Gateway
  • Access Control validates permissions
  • Session Manager loads context
  • Agent Runtime builds prompt
  • Model API processes request
  • Tool Executor runs tools if needed
  • Session saves state
  • Response streams back to user

Zustandsverwaltung

Wie OpenClaw persistente Daten speichert

  • Workspace — AGENTS.md, SOUL.md, TOOLS.md, skills/
  • Config — ~/.openclaw/openclaw.json
  • Environment Variables — override config
  • Credentials — ~/.openclaw/credentials/
  • Sessions — ~/.openclaw/sessions/
  • Memory — ~/.openclaw/memory/*.db
